# InterviewPrep AI — Frontend

> AI-powered interview preparation platform. Paste any job description, receive 5 tailored questions generated by Claude AI, answer them, and get instant expert feedback with a score out of 10.

**Live demo:** _coming soon_ · [Backend repo](https://github.com/koatth60/interview-prep-ai-back)

---

## Features

- **AI question generation** — Claude reads the job description and produces 5 role-specific questions mixing technical, behavioral, and situational angles
- **AI answer evaluation** — Every answer is scored 1–10 with detailed written feedback
- **Session history** — All past interviews saved with average score at a glance
- **Authentication** — Secure email/password login via NextAuth.js with JWT sessions
- **Fully typed** — TypeScript across the entire codebase

## Getting Started

### Prerequisites

- Node.js 18+
- PostgreSQL — [Neon](https://neon.tech) free tier is sufficient
- Anthropic API key — [console.anthropic.com](https://console.anthropic.com) (free $5 credit on signup)

### Installation

```bash
git clone https://github.com/koatth60/interview-prep-ai-front.git
cd interview-prep-ai-front
npm install
```

### Environment variables

Create `.env.local` at the project root:

```env
# PostgreSQL
DATABASE_URL="postgresql://user:password@host/dbname?sslmode=require"

# NextAuth — generate secret: openssl rand -base64 32
NEXTAUTH_SECRET="your-secret-here"
NEXTAUTH_URL="http://localhost:3000"

# Anthropic
ANTHROPIC_API_KEY="sk-ant-..."
```

### Database setup

```bash
npx prisma generate   # generate the Prisma client
npx prisma db push    # create tables from schema (no migrations needed)
```

### Run in development

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

## How It Works

```
1. User pastes a job description
         │
         ▼
2. POST /api/sessions
   └─ Claude generates 5 tailored questions
   └─ Session + questions saved to PostgreSQL
   └─ Redirect → /interview/[id]
         │
         ▼
3. User types an answer per question
         │
         ▼
4. POST /api/questions/[id]/answer
   └─ Claude evaluates the answer → { feedback, score }
   └─ Saved to DB, shown instantly in the UI
```

The entire AI logic lives in [`lib/claude.ts`](lib/claude.ts) — two functions, two prompts.

## Deployment (Vercel)
